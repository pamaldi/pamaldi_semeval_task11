# CRITICAL RULES

## 1. RULE DIRECTION - MOST IMPORTANT
"All S are P" or "Every S is a P" or "Anything that is S is P" ALWAYS encodes as:
p(X) <= s(X)

Read it as: "If X is an S, then X is a P"

Examples:
- "Every city is a location" → location(X) <= city(X)
- "All dogs are mammals" → mammal(X) <= dog(X)  
- "Anything that is a bird is an animal" → animal(X) <= bird(X)
- "Every cat is invisible" → invisible(X) <= cat(X)

NEVER write: s(X) <= p(X)  (this means the opposite!)

## 2. ALWAYS Define ALL Predicates
- Every predicate used MUST be in create_terms()
- Every predicate used in negation ~p(X) MUST have a fact: +p('_dummy')
- Every predicate in the conclusion's subject MUST have a fact

## 3. EXACT SPELLING - NO SHORTCUTS
Every predicate name in code must EXACTLY match create_terms().
- NEVER use placeholder names like 's', 'p', 'm' for predicates
- ALWAYS use the full predicate name from the syllogism
- If the syllogism says "capital cities", use capital_city, NOT s

## 4. VALID SYNTAX ONLY
Facts are ONLY written as: +predicate('value')
- CORRECT: +beverage('w')
- WRONG: +p(beverage('w'))  ← This is invalid syntax!
- WRONG: +s('w')  ← 's' is not a predicate!

## 5. WITNESS SHARING
For "Some S are P" (I-type premise), use the SAME witness for both:
```python
+s('w')
+p('w')  # Same 'w' - this entity is both S and P
```

## 6. Proposition Encoding Patterns

### A-type: "All S are P" / "Every S is a P"
```python
p(X) <= s(X)
```

### E-type: "No S is P" / "S are never P"
```python
+s('_s')  # Ensure s is defined
+p('_p')  # Ensure p is defined
conflict(X) <= s(X) & p(X)
```

### I-type: "Some S are P" / "A few S are P"
```python
+s('w')
+p('w')  # SAME witness
```

### O-type: "Some S are not P" / "Not all S are P"
```python
+s('w')
+p('_other')  # Define p but NOT for witness w
```

## 7. Conclusion Validity Checks - CHECK EXACTLY WHAT THE CONCLUSION STATES

### For A-conclusion "All S are P":
```python
+p('_dummy')  # Ensure p is defined for negation
invalid(X) <= s(X) & ~p(X)
result = invalid(X).data
print("VALID" if not result else "INVALID")
```

### For E-conclusion "No S is P":
```python
invalid(X) <= s(X) & p(X)
result = invalid(X).data
print("VALID" if not result else "INVALID")
```

### For I-conclusion "Some S are P":
Check EXACTLY the predicates in the conclusion:
```python
result = (s(X) & p(X)).data  # s and p from the CONCLUSION, not premises!
print("VALID" if result else "INVALID")
```

### For O-conclusion "Some S are not P":
```python
+p('_dummy')  # CRITICAL: Define p for negation
result = (s(X) & ~p(X)).data
print("VALID" if result else "INVALID")
```

## 8. INVALID Syllogism Strategy
For INVALID syllogisms (undistributed middle, illicit major/minor):
- Use DIFFERENT witnesses for different predicates
- This creates a model where premises are satisfied but conclusion fails

Example - AAI-2 (Invalid undistributed middle):
"All A are B. All C are B. Therefore, some C are A."
```python
+c('x')      # Witness for C
+a('y')      # DIFFERENT witness for A - no overlap!
b(X) <= a(X)
b(X) <= c(X)
result = (c(X) & a(X)).data  # Check if any C is also A
print("VALID" if result else "INVALID")  # INVALID - no overlap
```
