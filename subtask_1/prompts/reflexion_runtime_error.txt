# Reflexion Error Hints for NestedQueryError (v2 - CRITICAL FIX)

## STOP! READ THIS FIRST!

The error message `NestedQueryError: The last query was not closed` is **MISLEADING**.

❌ **WRONG DIAGNOSIS**: "The valid_syllogism rule is causing the error"
✅ **CORRECT DIAGNOSIS**: "A predicate is being redefined after it already has facts/rules"

The error occurs during `prolog.consult(filepath)` - that means the file LOADING is failing, not the query execution!

---

## THE ACTUAL CAUSE (READ CAREFULLY)

PySwip's `consult()` function crashes when it encounters **predicate redefinition patterns**. This happens when:

1. You define facts for a predicate
2. Then later define a `fail` clause or rule for THE SAME predicate

### Example of BROKEN code:

```prolog
% This code causes NestedQueryError:
fish(trout).           % Line 1: fish/1 is defined as a fact
fish(salmon).          % Line 2: another fish fact (OK)
% ... other code ...
fish(_) :- fail.       % Line 10: CRASH! fish/1 already exists!
```

### Example of WORKING code:

```prolog
% Option A: ONLY use facts (no fail clause)
fish(trout).
fish(salmon).
% DO NOT add fish(_) :- fail.

% Option B: ONLY use fail clause (no facts)
fish(_) :- fail.
% DO NOT add fish(trout). facts
```

---

## CRITICAL RULES - MEMORIZE THESE

### Rule 1: Choose ONE definition style per predicate

For each predicate, pick ONE:
- **Facts only**: `pred(value1). pred(value2).`
- **Fail clause only**: `pred(_) :- fail.`
- **Rules only**: `pred(X) :- other_pred(X).`

**NEVER MIX THEM FOR THE SAME PREDICATE NAME**

### Rule 2: No :- directives

These ALL cause NestedQueryError:
```prolog
:- dynamic pred/1.        % BANNED
:- discontiguous pred/1.  % BANNED
:- module(...).           % BANNED
:- initialization(...).   % BANNED
:- asserta(...).          % BANNED
:- valid_syllogism.       % BANNED
```

### Rule 3: Rules can coexist with facts

This is OK:
```prolog
cat(fluffy).              % fact
mammal(X) :- cat(X).      % rule using cat/1 (different predicate!)
```

This is NOT OK:
```prolog
cat(fluffy).              % fact for cat/1
cat(_) :- fail.           % fail clause for cat/1 - CRASH!
```

## HOW TO FIX YOUR CODE

### Step 1: Identify all predicates with facts

List every `predicate(value).` fact in your code.

### Step 2: Remove fail clauses for those predicates

If you have `bird(robin).` as a fact, DELETE any `bird(_) :- fail.` line.

### Step 3: Only use fail clauses for EMPTY predicates

```prolog
% Predicates WITH instances (use facts):
bird(robin).
fish(trout).

% Predicates WITHOUT instances (use fail clause):
mammal(_) :- fail.
reptile(_) :- fail.

% Rules are fine:
animal(X) :- bird(X).
animal(X) :- fish(X).
```

## COMMON MISTAKES TO AVOID

### Mistake 1: Adding fail clause "just to be safe"
```prolog
% WRONG - causes crash:
cat(fluffy).
cat(_) :- fail.   % NO! cat already has facts!
```

### Mistake 2: Defining predicate in multiple places
```prolog
% WRONG - causes crash:
fish(trout).
bird(robin).
fish(_) :- fail.  % NO! fish already defined above!
```

### Mistake 3: Using :- dynamic
```prolog
% WRONG - causes crash:
:- dynamic cat/1, dog/1.
cat(fluffy).
```

### Mistake 4: Using :- to run queries
```prolog
% WRONG - causes crash:
valid_syllogism :- bird(X), mammal(X).
:- valid_syllogism.   % NO! Don't run queries in the file!
```
