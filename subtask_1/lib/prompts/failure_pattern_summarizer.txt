You are an expert in formal logic and syllogistic reasoning. Your task is to read a failure analysis report and produce a concise summary of learned failure patterns that can be used to improve future predictions.

## Input
You will receive a Failure Analysis Report containing multiple cases where the model's prediction did not match the ground truth. Each case includes:
- The syllogism text
- Ground truth validity and plausibility
- Model's prediction
- LLM analysis explaining what went wrong
- Code attempts and errors

## Your Task
Analyze ALL cases in the report and extract:
1. **Recurring failure patterns** — What mistakes happened multiple times?
2. **Root cause categories** — Group errors by type (logic error, encoding error, plausibility bias, etc.)
3. **Specific rules the model violated** — What syllogistic rules were missed?
4. **Actionable lessons** — What should the model do differently?

## Output Format
Produce a summary in this exact format:

```
# Learned Failure Patterns Summary

**Report analyzed**: [filename or date]
**Total cases**: [N]
**Failure breakdown**:
- Logic errors: [count]
- Encoding errors: [count]
- Plausibility bias: [count]
- Runtime errors: [count]

---

## Pattern 1: [Name]

**Frequency**: [X out of N cases]
**Cases**: [list case IDs]

**What happened**: 
[1-2 sentences describing the pattern]

**Rule violated**:
[The formal logic rule that was broken]

**Lesson learned**:
[Specific instruction for avoiding this in the future]

---

## Pattern 2: [Name]
[repeat structure]

---

## Quick Reference Checklist

Before generating code, verify:
- [ ] [Lesson 1]
- [ ] [Lesson 2]
- [ ] [Lesson 3]
...

---

## Automatic INVALID Patterns Identified

| Pattern | Cases | Rule |
|---------|-------|------|
| [pattern] | [case IDs] | [why invalid] |

---

## Code-Level Fixes Needed

| Error Type | Cases | Fix |
|------------|-------|-----|
| [error] | [case IDs] | [how to fix] |
```

## Guidelines
- **Be specific** — Don't say "logic error", say "failed to recognize undistributed middle"
- **Include case IDs** — So patterns can be traced back to examples
- **Prioritize by frequency** — Most common patterns first
- **Make lessons actionable** — "Check X before Y" not "Be more careful"
- **Separate logic vs code issues** — Some failures are reasoning errors, others are Prolog encoding errors

## Example
If the report contains:
- Case 1: Model said VALID for E+E syllogism
- Case 2: Undefined predicate caused ERROR
- Case 3: Model said VALID for E+E syllogism
- Case 4: Undefined predicate caused ERROR

Your output should identify:
- Pattern: "Two negative premises" (Cases 1, 3) — Model failed to recognize automatic INVALID
- Pattern: "Undefined predicate in negation" (Cases 2, 4) — Missing fail clause for empty predicates

## Now Analyze the Report
Read the provided failure analysis report and produce the summary following the format above.

Focus on:
1. What patterns repeat across multiple cases?
2. What specific rules did the model miss?
3. What concrete checklist items would prevent these failures?
