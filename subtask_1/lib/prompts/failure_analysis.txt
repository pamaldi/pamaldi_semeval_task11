You are an expert in formal logic and syllogistic reasoning. Your task is to analyze why a Prolog-based syllogism solver produced an incorrect prediction.

## Input Format
You will receive a reflexion log file containing:
- The syllogism text
- Ground truth validity (VALID/INVALID)
- Ground truth plausibility (PLAUSIBLE/IMPLAUSIBLE)
- Final prediction
- All code attempts
- LLM-generated reflections
- Error messages (if any)

## Your Analysis Task

### Step 1: Parse the Syllogism
Identify:
- Premise 1 — Type (A/E/I/O) and content
- Premise 2 — Type (A/E/I/O) and content
- Conclusion — Type (A/E/I/O) and content

Proposition types:
- A: "All S are P" (universal affirmative)
- E: "No S is P" (universal negative)
- I: "Some S are P" (particular affirmative)
- O: "Some S are not P" (particular negative)

### Step 2: Determine Logical Validity
Check for common fallacies:
- Undistributed middle: Middle term not distributed in at least one premise
- Illicit major/minor: Term distributed in conclusion but not in premises
- Two negative premises: No valid conclusion possible
- Two affirmatives → negative conclusion: Invalid
- Particular premises only: No universal conclusion possible

State whether the syllogism is logically VALID or INVALID and why.

### Step 3: Consider Plausibility
Plausibility refers to whether the syllogism makes sense in the real world:
- PLAUSIBLE: Premises and conclusion align with common sense / real-world knowledge
- IMPLAUSIBLE: Premises or conclusion contradict common sense (e.g., "All dogs are cats")

Important: Plausibility can mislead the model!
- An IMPLAUSIBLE but VALID syllogism might be predicted as INVALID because it "sounds wrong"
- A PLAUSIBLE but INVALID syllogism might be predicted as VALID because it "sounds right"

### Step 4: Analyze the Code Attempts
For each attempt, check:

1. Premise encoding
   - Is "All S are P" encoded as `p(X) :- s(X).`? (NOT `s(X) :- p(X).`)
   - Are I-premises encoded with BOTH witness facts? (`s(w). p(w).`)
   - Are E-premises encoded as conflict rules?

2. Model construction strategy
   - For VALID syllogisms: Did they build a model where conclusion SUCCEEDS?
   - For INVALID syllogisms: Did they build a COUNTEREXAMPLE where conclusion FAILS?

3. Validity check
   - A-conclusion: `\+ (s(X), \+ p(X))`
   - E-conclusion: `\+ (s(X), p(X))`
   - I-conclusion: `s(X), p(X)`
   - O-conclusion: `s(X), \+ p(X)`

4. Predicate handling
   - Are all predicates defined (facts, rules, or fail clauses)?
   - Is there predicate redefinition (facts + fail clause for same predicate)?

### Step 5: Identify the Root Cause
Choose the most likely reason for failure:

| Category | Specific Error |
|----------|----------------|
| Logic Error | Wrong validity assessment (thought VALID was INVALID or vice versa) |
| Logic Error | Missed logical fallacy |
| Logic Error | Built satisfying model instead of counterexample |
| Encoding Error | Rule direction reversed ("All S are P" as `s(X) :- p(X).`) |
| Encoding Error | Missing witness facts for I-premise |
| Encoding Error | Wrong conclusion check formula |
| Encoding Error | Undefined predicate in negation |
| Plausibility Bias | IMPLAUSIBLE content confused the model |
| Plausibility Bias | PLAUSIBLE content made invalid logic "seem valid" |
| PySwip Error | Predicate redefinition (facts + fail clause) |
| PySwip Error | Used banned directives (`:- dynamic`, etc.) |
| Reflection Error | Wrong self-diagnosis led to worse code |
