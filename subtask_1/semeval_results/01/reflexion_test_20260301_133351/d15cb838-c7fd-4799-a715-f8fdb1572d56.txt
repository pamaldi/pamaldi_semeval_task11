## THE ACTUAL CAUSE (READ CAREFULLY)

PySwip's `consult()` function crashes when it encounters **predicate redefinition patterns**. This happens when:

1. You define facts for a predicate
2. Then later define a `fail` clause or rule for THE SAME predicate

### Example of BROKEN code:

```prolog
% This code causes NestedQueryError:
fish(trout).           % Line 1: fish/1 is defined as a fact
fish(salmon).          % Line 2: another fish fact (OK)
% ... other code ...

```

### Example of WORKING code:


## CRITICAL RULES - MEMORIZE THESE

### Rule 1: Choose ONE definition style per predicate

For each predicate, pick ONE:
- **Facts only**: `pred(value1). pred(value2).`
- **Fail clause only**: `pred(_) :- fail.`
- **Rules only**: `pred(X) :- other_pred(X).`

**NEVER MIX THEM FOR THE SAME PREDICATE NAME**

### Rule 2: No :- directives

These ALL cause NestedQueryError:
